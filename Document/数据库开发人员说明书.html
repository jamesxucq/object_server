Object Server开发人员说明书

 

一.样例数据库搭建

    运行后面的例子时不要复制粘贴.编码不一样.

1.ObjectServer安装完成后,执行以下命令启动ObjectServer服务器:

    james@ubuntu:~$/opt/object_server/bin/osev_serv_noau

osev_serv_noau为没有权限模块的测试版,在前台运行需要占用一个控制台.osev_serv_t是发行版,后台运行.

2.服务器启动完成后,执行以下命令启动ObjectServer客户端:

    james@ubuntu:~$/opt/object_server/bin/osev-cli

客户端输入状态下按方向键上下可以翻看输入历史.运行后面的例子时不要复制粘贴.编码不一样.

3.客户端执行以下命令,连接服务器.

    clie: conne osvp://127.0.0.1:8848/?uname=xman&paswd=*****

注意修改相应的服务器ip,用户名和密码.

4.客户端执行command对象的open方法,打开earth数据库.

    osev: command:open(earth);

如果earth数据库存在就打开,不存在就创建.

执行以下对象的方法,进入earth数据库.

    osev: clie:use(earth, dev1, 123abc);

数据库连接成功后提示符变成"odb-lv4p> ",说明已经成功进入earth数据库的第四层空间.执行clie:level方法进入数据库的其他层空间.

    odb-lv4p> clie:level(2p);

用上面的方法进入数据库二层空间,输入下面的txxt类定义,在数据库中声明txxt这个类.

    odb-lv2p> txxt<[xx int32, ax int32, cx int32], tpp(cx, ax, xx)>;

执行clie:level方法进入txxt类的一层空间.

    odb-lv2p> clie:level(txxt);

输入下面的txxt:tpp方法的定义,在数据库中声明txxt:tpp这个方法.

    olv1.txxt> :tpp<(xo int32, ao int32)int32, (ci int32, ai int32, xi int32)> {xi = xo; ai = ao; return ci;}

把客户端切换到数据库第四层空间.

    olv1.txxt> clie:level(4p);

以下调用在数据库中创建"txxt_con"这个容器.

    odb-lv4p> witi:witi("txxt_con", "txxt");

执行clie:level方法进入txxt_con的三层空间.

    odb-lv4p> clie:level(txxt_con);

在txxt_con这个容器中创建三个对象,输入如下:

    txxt_con-lev3> txxt:txxt(155,100,222);

    txxt_con-lev3> txxt:txxt(166,210,322);

    txxt_con-lev3> txxt:txxt(366,410,522);

执行以下调用,查看创建的对象

    txxt_con-lev3> :search() | kits:pretty("oiii");

 

二.cplusplus样例说明

1.简要说明

    在源码仓这个地址: NetBeansProjects/client/cplusplus/Example可以看到一些netbeans的工程,他们分别介绍了cpp怎么和ObjectServer进行对象增删改查询的.如果没有安装netbeans,可以进入工程目录make编译.之后进入./dist/Debug/GNU-Linux这个目录是示例可执行文件.

2.从数据库导出类的声明和定义,给cpp程序使用.

    进入数据库二层空间.下面的调用在本地目录生成txxt类的方法的定义.

    odb-lv2p> kits:list(txxt, LEVO, FORCED) | kits:translate(GCC) | kits:save(txxt.cpp);

    下面的调用在本地目录生成txxt类的方法的定义.

    odb-lv2p> kits:list(txxt, LVTP, FORCED) | kits:translate(GCC) | kits:save(txxt.h);

3.本地目录结构和文件的介绍

    NetBeansProjects/client/cplusplus/Example这目录放的cpp示例文件,此目录下的earth_lv2p目录是earth这个数据库的class类文件定义

txxt.cpp ,txxt.h这两个文件是txxt类的cpp定义.EarthLvtpHeader.h这个文件是cpp编译的一些功能选项和头文件声明的集合.

    功能选项解释如下:

    #ifndef CHECK_ATTRIBU_DECLARE

    #define CHECK_ATTRIBU_DECLARE

系统自动检测服务器端类定义变更的频率

启用_CHECK_VERSION_INTERVAL和_CHECK_ATTRIBU_INTERVAL需使用这个宏

    #define CHECK_ATTRIBU_FREQUENCY   (1024-1) // 2^10-1

下面的宏是五种检测服务器类变化方式选项

check server and local lv2p attribute: version, external

每次都检测类版本的变化

    #define _CHECK_VERSION_EVERYTIME

按上面的频率检测类版本的变化

    #define _CHECK_VERSION_INTERVAL // interval

每次都检测类版本和外联属性的变化

    #define _CHECK_ATTRIBU_EVERYTIME

按上面的频率检测类版本的变化和外联属性

    #define _CHECK_ATTRIBU_INTERVAL // interval

不检测类版本的变化和外联属性

    #define _DONOT_CHECK_ATTRIBU
